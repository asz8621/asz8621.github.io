認識 AJAX，了解同步與非同步與 HTTP 狀態碼

發表於 2024-09-25 分類於 JavaScript
深入了解 AJAX，包括如何使用 XMLHttpRequest 物件建立請求、獲取伺服器資料並在瀏覽器中進行渲染，提供兩種 AJAX 傳送方式的範例程式碼，分別展示傳統表單提交與 JSON 格式的資料傳送，並說明如何根據後端需求選擇適合的傳送方式。

使用流程

  1. 建立 AJAX 物件
  2. 發送請求到伺服器獲取資料
  3. 伺服器回傳資料到瀏覽器
  4. 獲取資料並進行渲染或加工處理

1. 建立 XMLHttpRequest 物件

使用 XMLHttpRequest 物件來進行 AJAX 請求，並在 console 中查看 xhr 變數的內容。

重點：readyState 的數字代表不同的狀態

readyState 數字 代表涵義
0 XMLHttpRequest 物件已建立，但尚未連結
1 已調用 open()，但尚未發送請求
2 偵測到已調用 send()
3 正在加載中，可能資料量較大
4 資料已完全獲取

2. 使用 open() 連結資料

open() 方法用來設定請求的格式、網址以及同步或非同步。

格式 代表涵義
GET 讀取資料
POST 發送資料到伺服器
同步與非同步 代表涵義
true 非同步，不會等待回應，程式碼會繼續執行
false 同步，會等待回應後再繼續執行

由於調用了 open()，此時 readyState 為 1。

3. 使用 send() 傳送資料

我們目前只需讀取資料，因此可傳送 null。

  • 使用 send() 後，readyState 會從 2（偵測到 send()）變為 3（加載中），最終變為 4（資料獲取完成）。

HTTP 狀態碼

狀態碼 代表涵義
200 成功 (OK)
3XX 重新導向 (Redirection)
404 找不到檔案或目錄

Cross-Origin Resource Sharing (CORS)

CORS 是一種機制，使用額外的 HTTP 標頭使得瀏覽器能夠獲取來自不同來源的資源。當請求的資源來自不同的網域、協定或通訊埠時，會建立一個跨來源 HTTP 請求。

因為安全性考量，跨來源請求受到限制。XMLHttpRequest 和 Fetch 都遵循同源政策（Same-Origin Policy），除非使用 CORS 標頭，否則只能請求與應用程式相同網域的資源。

簡而言之，在同一網域下 API 可以自由使用，但在不同網域下必須確認對方的後端是否開啟 CORS，否則會出現錯誤訊息。
